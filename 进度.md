# Geo 协作发布平台进度

更新时间：2026-05-11

## 总体判断

当前整体进度略快于原排期。5月7日-5月11日的本地 MVP 与基础工程能力已经基本完成；5月11日-5月15日的云端浏览器发布能力已经完成关键 Spike，并开始正式后端化。

## 已完成

### 本地 MVP 与基础工程能力

- FastAPI、React、SQLite/Alembic、本地数据目录和启动器已跑通。
- 文章、资源、账号、任务、发布记录、日志等核心模块已有基础实现。
- 本地 exe MVP 已完成过构建验证。
- Alembic 迁移链路可用。
- 头条发布器已接入任务执行主链路。

### 头条号发布 Spike

- 已验证头条号登录态保存和复用。
- 已跑通发布页进入、标题填充、正文填充、封面上传、正文图片上传。
- 已验证发布预览链路。
- 已完成一次头条真实发布成功验证。
- 已处理部分营销/引导弹窗的脚本关闭逻辑。

### 云服务器验证

- 阿里云 Ubuntu 22.04.5 环境已完成基础验证。
- Chromium、Xvfb、x11vnc、websockify/noVNC 已能运行。
- 本地浏览器可通过 SSH 隧道访问 noVNC。
- 云端扫码登录和发布页操作已验证。
- 云端安全发布 Smoke 已跑通，能进入发布预览并等待确认。
- 云端真实发布链路已经验证成功。

### 后端能力补强

- 发布浏览器 channel、executable path、并发数已配置化。
- 发布并发上限已按 5 个记录控制。
- 已加入 `waiting_user_input` 状态，用于扫码、验证码、登录失效等人工介入场景。
- 已加入人工介入恢复接口，处理完成后可重新入队。
- 任务状态机已覆盖人工确认、人工介入、失败、恢复等场景。
- Python 3.10 兼容问题已修复。
- 已新增远程浏览器会话服务，后端可配置托管 Xvfb、x11vnc、websockify/noVNC。
- Playwright 发布入口已接入远程浏览器会话，云端开启配置后可自动分配 display、VNC 端口和 noVNC 端口。
- 系统状态接口已暴露远程浏览器开关、就绪状态和活跃会话数量。

## 测试情况

- 浏览器会话管理测试通过。
- 头条发布器入口相关测试通过。
- 系统状态接口测试通过。
- 任务状态机测试通过。
- 删除保护、模型、账号、文章、资源、发布校验等受影响测试通过。
- FTS 和迁移测试通过。
- 完整 `server/tests` 曾受旧 launcher 测试耗时影响超时，非本轮改动直接失败点。

## 当前缺口

- 远程浏览器会话已后端化，但还没有前端入口展示 noVNC 链接。
- `waiting_user_input` 进入人工介入时，当前发布器仍会关闭浏览器；下一步要让人工介入会话可保留并被用户打开。
- 云端账号鉴权还没做，当前仍偏本地单用户模式。
- 数据库还没有从 SQLite 切换到 MySQL。
- 飞书通知还没有接入真实 API。
- 管理后台的任务、日志、截图、人工介入体验还需要补齐。
- 压测和长时间稳定性验证还没开始。
