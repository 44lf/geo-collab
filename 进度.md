# Geo 协作发布平台进度

## 总体判断

当前整体进度略快于原排期。第一阶段和第二阶段已全部完成，可以按计划进入第三阶段（5/16-5/22）的准备工作。

## 已完成

### 本地 MVP 与基础工程能力

- FastAPI、React、SQLite/Alembic、本地数据目录和启动器已跑通。
- 文章、资源、账号、任务、发布记录、日志等核心模块已有基础实现。
- 本地 exe MVP 已完成过构建验证。
- Alembic 迁移链路可用。
- 头条发布器已接入任务执行主链路。

### 头条号发布 Spike

- 已验证头条号登录态保存和复用。
- 已跑通发布页进入、标题填充、正文填充、封面上传、正文图片上传。
- 已验证发布预览链路。
- 已完成一次头条真实发布成功验证。
- 已处理部分营销/引导弹窗的脚本关闭逻辑。

### 云服务器验证

- 阿里云 Ubuntu 22.04.5 环境已完成基础验证。
- Chromium、Xvfb、x11vnc、websockify/noVNC 已能运行。
- 本地浏览器可通过 SSH 隧道访问 noVNC。
- 云端扫码登录和发布页操作已验证。
- 云端安全发布 Smoke 已跑通，能进入发布预览并等待确认。
- 云端真实发布链路已经验证成功。

### 后端能力补强

- 发布浏览器 channel、executable path、并发数已配置化。
- 发布并发上限已按 5 个记录控制。
- 已加入 `waiting_user_input` 状态，用于扫码、验证码、登录失效等人工介入场景。
- 已加入人工介入恢复接口，处理完成后可重新入队。
- 任务状态机已覆盖人工确认、人工介入、失败、恢复等场景。
- Python 3.10 兼容问题已修复。
- 已新增远程浏览器会话服务，后端可配置托管 Xvfb、x11vnc、websockify/noVNC。
- Playwright 发布入口已接入远程浏览器会话，云端开启配置后可自动分配 display、VNC 端口和 noVNC 端口。
- 系统状态接口已暴露远程浏览器开关、就绪状态和活跃会话数量。
- noVNC 前端入口完成：`waiting_user_input` 状态的记录可展示"打开远程浏览器"按钮，点击在新标签页打开 noVNC。
- 浏览器保留机制完成：`ToutiaoUserInputRequired` 时上下文管理器不再关闭浏览器，远程会话标记为 keep-alive。
- `PublishRecord` 响应体新增 `remote_browser_session_id` 和 `novnc_url` 字段。
- 云端部署文档 `docs/deploy.md` 完成：环境变量清单、启动步骤、架构说明。
- 远程浏览器空闲超时自动清理：`_session_keep_alive` 超 5 分钟后自动 stop 并解除 record 关联。
