# Geo 协作平台实施计划

本目录用于承接 `Geo协作平台-设计文档.md` 的落地实施。当前计划按“一周一个可验收目标”重新拆分：先看排期和本周任务，再按需查看模块任务细节。

## 工作原则

- 目标是在一个月内完成可本地运行、可演示、可试用的 Windows MVP。
- 首期核心价值不是做完整后台，而是打通：文章编辑 -> 账号授权 -> 任务创建 -> 头条号半自动填充 -> 人工发布确认 -> 结果记录。
- 任务优先级按 `P0/P1/P2`：
  - `P0`：没有它 MVP 不成立。
  - `P1`：影响试用体验，但可以降级。
  - `P2`：增强项，可延期。
- 所有任务状态必须在 `progress.md` 更新。
- 完成任务时必须补充验证方式，包括命令、手工步骤或截图说明。
- 遇到头条号页面变化、登录风控、打包失败等阻塞，应更新 `progress.md` 的阻塞说明，不要静默绕过。

## 状态定义

| 状态 | 含义 |
| --- | --- |
| Not Started | 尚未开始 |
| In Progress | 正在实现 |
| Blocked | 被外部条件或重大技术问题阻塞 |
| Review | 已实现，等待人工或另一个 AI 检查 |
| Done | 已实现并验证 |
| Deferred | 明确延期到 MVP 之后 |

## 接手顺序

1. 阅读 `Geo协作平台-设计文档.md`。
2. 阅读本文件。
3. 查看 `schedule.md`，确认当前处于哪一周、目标是什么。
4. 查看 `progress.md` 当前状态。
5. 打开对应 `week-*.md`，选择一个未开始且依赖已满足的 `P0` 任务。
6. 开始前把 `progress.md` 中该任务状态改为 `In Progress`。
7. 完成后把验证结果写入对应 `week-*.md` 和 `progress.md`。

## 任务拆分方式

- `week-*.md` 是执行主线，按排期组织，优先看这里。
- `tasks-*.md` 是模块视图，用于查接口范围、模型范围和局部验收点。
- 一个周任务可能覆盖多个模块任务，例如 Week 1 的“项目可启动”同时覆盖 `BE-001`、`FE-001`、`PK-002` 的一部分。
- 如果周任务和模块任务描述冲突，以周任务的交付目标为准，再把差异记录到 `progress.md`。
