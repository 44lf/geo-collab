# Geo 协作平台 — 交付说明

版本：0.1.0  
适用系统：Windows 10 / 11 x64

---

## 前置条件

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10 或 11 64 位 |
| Google Chrome | 必须已安装（用于头条号浏览器自动化） |
| 网络 | 首次登录头条号需要访问互联网 |

不需要安装 Python、Node.js、Docker 或数据库。

---

## 启动

1. 双击 `GeoCollab.exe`。
2. 等待约 2–3 秒，默认浏览器会自动打开 `http://127.0.0.1:87xx`。
3. 关闭浏览器标签页不会停止服务；关闭终端窗口才会退出。

> 数据保存在 `C:\Users\<你的用户名>\AppData\Local\GeoCollab\`（重启后保留）。

---

## 使用流程

### 1. 添加头条号账号

1. 点击左侧导航 **媒体矩阵**。
2. 填写「显示名称」和「本地状态目录」（目录名只是本地标识，随意填）。
3. 点击 **添加授权** — 会打开 Chrome 浏览器，请在页面内完成登录，等待系统检测到登录成功。
4. 下次打开时点击 **复用状态** 即可跳过登录。

### 2. 新建文章

1. 点击左侧导航 **内容管理**。
2. 点击右上角 **新建图文**，填写标题、作者、正文。
3. 可上传封面图片和正文图片。
4. 状态设为 **待发布**，点击 **保存**。

### 3. 创建文章分组（可选，用于批量发布）

1. 在内容管理左侧勾选多篇文章。
2. 填写分组名称，点击 **创建**。

### 4. 发布任务

1. 点击左侧导航 **分发引擎**。
2. 点击 **创建任务**：
   - **单篇发布**：选择一篇文章 + 一个账号。
   - **分组轮询**：选择一个分组 + 多个账号，系统自动轮询分配。
3. 点击 **预览分配** 确认文章与账号的对应关系（仅分组任务）。
4. 点击 **创建任务**，然后在右侧详情里点击 **执行**。
5. 系统会自动打开 Chrome 填充标题、正文、封面，然后停在「发布」按钮前。
6. 在浏览器里手动点击发布，回到 Geo 点击 **标记成功**（可填写已发布链接）。
7. 若填充失败，可点击 **标记失败** 后点击 **重试**。

### 5. 导出授权包

在 **媒体矩阵** 页面点击 **导出授权包**，下载 zip 文件用于备份或迁移。

## 数据位置

| 内容 | 路径 |
|------|------|
| 数据库 | `%LOCALAPPDATA%\GeoCollab\geo.db` |
| 图片资产 | `%LOCALAPPDATA%\GeoCollab\assets\` |
| 浏览器状态 | `%LOCALAPPDATA%\GeoCollab\browser_states\` |
| 运行日志 | `%LOCALAPPDATA%\GeoCollab\logs\launcher.log` |
| 导出文件 | `%LOCALAPPDATA%\GeoCollab\exports\` |

## 已知问题

| 编号 | 问题 | 临时处理 |
|------|------|---------|
| K-01 | 开启系统代理时头条号登录可能出现验证码卡住 | 关闭代理后重新登录 |
| K-02 | 发布结果（链接）需手动填写，尚未自动检测 | 在浏览器复制链接后粘贴到「标记成功」输入框 |
| K-03 | exe 启动后会显示终端窗口 | 正常现象，可最小化；正式版将隐藏 |
| K-04 | 首次打开 exe 可能触发 Windows 防病毒提示 | 点击「仍然运行」即可 |

## 系统状态检查

点击左侧导航 **系统状态** 可查看：
- 服务运行状态
- Chrome 是否检测到
- 文章 / 账号 / 任务数量
- 数据目录路径
